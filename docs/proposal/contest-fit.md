# 오늘서울 공모 적합성 메모

## 심사 기준 대응

### 1. 공공데이터 활용성

- 서울 열린데이터광장 데이터를 후보군으로 사용
- 공공데이터포털 데이터를 당일 컨디션 판단에 사용
- 두 포털의 역할이 명확하게 분리되어 설명력이 높음

### 2. 시민 체감 가치

- 서울 시민이 자주 하는 질문인 “오늘 어디 가면 좋지?”에 답함
- 지역구, 동행자, 시간대, 예산, 이동 범위를 반영함
- 단순 조회가 아니라 실제 방문 또는 예약으로 연결함

### 3. 사업성

- 시민용 추천 서비스로 바로 확장 가능
- 자치구 문화·복지·청년 정책 안내 서비스로도 전환 가능
- 향후 혼잡도, 교통, 안전 데이터 결합 시 생활형 플랫폼으로 확장성 높음

### 4. 구현 완성도

- 프론트엔드와 백엔드가 분리된 실제 동작 구조
- 추천 결과에 출처, 이유, 실행 체크리스트 포함
- 발표 시 2분 내 데모 가능한 흐름 보유

## 본선에서 강한 포인트

- “포털 2개를 썼다”가 아니라 “후보 데이터와 판단 데이터를 나눠 썼다”는 설계 논리가 있음
- 서울 시민 사용 장면이 구체적이라 심사위원이 바로 이해할 수 있음
- 추천 이유와 데이터 출처가 화면에 함께 드러나 발표 설득력이 높음
