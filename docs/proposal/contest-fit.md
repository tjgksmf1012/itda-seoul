# 잇다서울 공모 적합성 메모

## 왜 이 대회와 맞는가

### 1. 공공데이터 2개 포털을 명확히 사용한다

- 서울 열린데이터광장
  - 복지시설
  - 문화행사
  - 공공서비스예약
- 공공데이터포털
  - 기상청 예보
  - 대기질 정보

두 포털의 역할을 섞지 않고 분리했다.

- 열린데이터광장: 행동 후보군 구성
- 공공데이터포털: 오늘의 이동 가능성 판단

### 2. AI 활용 지점이 분명하다

AI는 단순 문구 생성만 하지 않는다.

- 사용자 상태 해석
- 외출/참여/대화/보호자 처방 분류
- 후보 우선순위 결정
- 보호자 공유 문구 생성

즉 추천의 뼈대는 데이터, 개인화와 설명은 AI가 담당하는 하이브리드 구조다.

### 3. 서울시 정책 과제와 직접 연결된다

잇다서울은 일반 여가 추천 서비스가 아니라 고립위험 1인가구와 고령층의 첫 외출, 첫 참여, 첫 연결을 낮은 부담으로 시작하게 만드는 서비스다.

이 때문에 서울시의 외로움·고립 대응 정책과 문제정의가 자연스럽게 연결된다.

### 4. 본선 발표에 적합한 흐름이 나온다

발표는 아래 시나리오로 2분 안에 구성 가능하다.

1. 사용자 상태 입력
2. AI 사회연결 처방 결정
3. 복지/문화/예약 후보 3개 제시
4. 보호자 공유 문구 출력
5. 기관 운영 모드 확장 설명

## 심사위원 설득 포인트

- 데이터가 실제 행동 추천으로 바뀐다.
- 추천에서 끝나지 않고 신청과 공유까지 이어진다.
- 복지 현장 도입 시나리오가 명확하다.
- 서울시 데이터가 시민 체감 서비스로 전환된다.
