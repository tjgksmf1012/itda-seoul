# 잇다서울 공모전 적합성 메모

## 필수 요건 대응

- 서울 열린데이터광장 데이터 1건 이상 활용
- AI 기술 활용
- 아이디어 기획을 넘어 실제 서비스 MVP 화면과 API 구조 제시

## 가점 포인트 대응

- 인구·복지 데이터: 독거노인 현황
- 복지 인프라 데이터: 노인여가복지시설
- 행정 예약 데이터: 공공서비스예약 정보
- 문화 데이터: 서울시 문화행사 정보

위 네 축을 결합해 이종 데이터 결합 구조를 명확히 보여줄 수 있다.

## 심사 관점에서 강한 포인트

- 단순 정보 검색이 아니라 “다음 행동 처방”으로 문제를 재정의했다.
- 추천에서 끝나지 않고 신청 링크, 보호자 공유, 기관 체크인까지 연결된다.
- 개인 사용자 흐름과 기관 운영 화면을 함께 보여줘 실제 도입 가능성이 높다.
- 고립위험 1인가구와 고령층이라는 사회문제 해결성과 서울시 정책 방향의 연결성이 높다.
