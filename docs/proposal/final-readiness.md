## 1. 공식 조건 정리

공고문 기준으로 창업 부문에서 확인된 필수 조건은 다음이다.

- 서울 열린데이터광장 데이터셋 `1건 이상` 활용
- `AI 기술(STT, 생성형 AI 등)` 활용
- 제품·서비스 개발 부문은 `시제품이 완료된 상태`

제출 안내 파일 기준으로 추가 확인된 제출 조건은 다음이다.

- 참가신청서, 개인정보 수집 및 이용동의서는 `PDF`로 제출
- 상세기획서는 `PPT 또는 PDF` 제출 가능
- 대표이미지는 `로고 이미지`여야 함
- 화면이미지와 결과물 링크가 필요함
- 활용 공공데이터 목록은 `포털명 + 데이터셋 명칭을 정확하게` 적어야 함

## 2. 가점 해석

현재 확보한 공식 문서 텍스트에서는 `공공데이터포털을 같이 쓰면 가점`이라는 문장이 직접적으로 확인되지는 않았다.

다만 아래 해석은 충분히 안전하다.

- `서울 열린데이터광장 + 공공데이터포털`을 함께 쓰는 것은 필수는 아니지만 강한 차별점이다.
- 특히 `복지 + 문화 + 예약 + 날씨/대기질`처럼 서로 다른 분야 데이터를 결합하면 심사 설득력이 확실히 올라간다.
- 제출 안내에서도 두 포털을 모두 사용한 경우 각각의 데이터셋 명칭을 정확히 적도록 되어 있어, 다중 포털 활용은 분명히 긍정 요소다.

따라서 발표와 문서에서는 이렇게 표현하는 것이 안전하다.

> 필수 요건은 서울 열린데이터광장 1건 이상 활용과 AI 활용이며, 잇다서울은 여기에 더해 공공데이터포털의 실시간 도시 시그널을 결합해 서로 다른 분야 데이터를 실제 서비스 흐름으로 통합했다.

## 3. 잇다서울이 이미 충족한 것

- 생성형 AI 연동을 통한 보호자 안내문·복지사 안내문 생성
- 사용자 화면 + 기관 운영 화면 + 추천 API + 이력 저장 흐름 구현

즉 현재 프로젝트는 `공모전형 프로토타입` 수준은 이미 넘겼다.

## 8. 발표에서 반드시 말할 문장

- 잇다서울은 서울 열린데이터광장 데이터를 기반으로 실제 참여 가능한 복지·문화·예약 후보를 구성한다.
- 공공데이터포털 데이터는 당일 외출 가능성과 이동 부담을 조정하는 실시간 시그널로 사용한다.
- 생성형 AI는 사용자 상태 해석, 보호자 공유 문구, 복지사 후속 안내문 생성에 사용된다.
- 따라서 본 서비스는 `서울 공공데이터 + 생성형 AI + 실제 행동 전환`을 모두 갖춘 창업 부문형 결과물이다.
