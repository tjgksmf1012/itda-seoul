# 2026 서울시 빅데이터 활용 경진대회 제출 계획

이 문서는 사용자가 제공한 공고문 `붙임2.2026+서울시+빅데이터+활용+경진대회+공고문(안)_통합본_v2.1.hwpx`와
참가신청 양식 zip을 기준으로, `잇다서울`의 제출 준비 상태와 남은 작업을 정리한 문서다.

## 1. 공식 요구사항 요약

### 창업 부문 필수 조건

- 서울 열린데이터광장 데이터셋 1건 이상 활용
- AI 기술(STT, 생성형 AI 등) 활용
- 제품·서비스 개발 부문은 시제품이 완료된 상태여야 함

### 제출 서류

- 참가신청서
- 개인정보 수집 및 이용 동의서
- 각서
- 재학(휴학)증명서: 학생일 경우
- 결과물 링크
- 상세기획서: PPT 20매 내외, PDF 제출도 가능
- 국세청 발급 사실증명원: 사업자등록사실여부
- 대표 이미지: 로고
- 화면 이미지

### 제출 형식 주의사항

- 참가신청서, 동의서, 재학증명서는 PDF 제출 필수
- 상세기획서는 PPT 또는 PDF 제출 가능
- 대표 이미지는 화면 캡처가 아니라 로고여야 함
- 활용한 공공데이터 목록은 포털별 데이터셋 명칭을 정확하게 기재해야 함

### 일정

- 접수 마감: 2026년 5월 13일(수) 18시
- 1차 결과 발표: 2026년 5월 22일(금)
- 2차 결과 발표: 2026년 6월 4일(목)
- 최종 경연 및 시상식: 2026년 7월 6일(월)

## 2. 현재 프로젝트 상태

### 이미 충족한 항목

- 웹 기반 시제품 존재
- 서울 열린데이터광장 + 공공데이터포털 2개 축을 제품 구조에 반영
- 추천 화면, 기관 운영 화면, 보호자 공유 문구 흐름 구현
- 백엔드 API와 프론트 빌드 가능
- 기관 운영 데이터 저장 계층 초안 구현

### 아직 부족한 항목

- AI 활용이 아직 심사위원에게 강하게 보일 정도로 명확하지 않음
- 실제 수집된 서울 열린데이터광장 데이터가 발표에서 바로 보이는 수준까지 연결되지 않음
- 추천 이후 참여 여부와 재추천 루프가 완성되지 않음
- 제출용 로고, 화면 이미지, 결과물 링크 운영 주소가 아직 정리되지 않음
- 상세기획서 20매 내외 구조가 아직 완성되지 않음
- 데이터셋 명칭을 공식 명칭 기준으로 정리한 제출용 표가 아직 없음

## 3. 우선순위 계획

### 1순위: AI 활용을 실제 기능으로 고도화

- OpenAI API 연동
- 사용자 상태 해석 문장 생성
- 보호자 공유 문구 생성
- 복지사 체크인 문구 생성

목표:
- 심사위원이 "규칙 기반 추천"이 아니라 "공공데이터 + AI 해석"으로 인식하게 만들기

### 2순위: 실데이터 시연 강화

- 서울 열린데이터광장 raw 수집 결과를 `programs.live.json`으로 생성
- 공공데이터포털 시그널 데이터와 함께 실제 반영 상태를 화면에서 표시
- 데이터셋명, 갱신 시각, 반영 건수를 발표 화면에 유지

목표:
- "샘플이 아니라 실제 서울 데이터 기반"이라는 근거 확보

### 3순위: 재추천 루프 완성

- 추천 생성 이력 저장
- 보호자 공유 이력 저장
- 참여 여부 기록
- 미참여 시 재추천 사유 생성

목표:
- 기관용 SaaS 설득력 강화

### 4순위: 제출 패키지 제작

- 참가신청서 초안 작성
- 상세기획서 20매 내외 구성
- 로고 제작
- 화면 이미지 캡처 세트 제작
- 데이터셋 목록 표 정리
- 결과물 링크 고정

목표:
- 접수 직전 문서 작업으로 흔들리지 않게 준비
