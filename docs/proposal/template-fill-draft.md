## 2. 참가신청서 핵심 문구

### 문제 정의

서울시는 외로움 없는 서울 정책을 통해 외로움과 고립을 구조적 사회문제로 다루고 있지만, 시민이 실제로 활용하는 서비스는 여전히 분절적이다. 독거노인 현황, 노인여가복지시설, 공공서비스예약, 문화행사 정보는 각각 존재하지만, 개인의 상황을 해석해 오늘 무엇을 하면 되는지 제안하고 신청과 공유까지 연결하는 서비스는 부족하다.

### 해결 방안

잇다서울은 서울 열린데이터광장과 공공데이터포털 데이터를 결합해, 고립위험 1인가구와 고령층에게 지금 가장 부담이 낮은 다음 연결 행동을 AI가 추천하는 사회연결 처방 플랫폼이다. 사용자의 외출 가능 정도, 도보 가능 거리, 최근 외출 빈도, 보호자 개입 여부, 관심사를 입력받으면, AI가 외출형·참여형·대화형·보호자형 처방 중 가장 적합한 흐름을 선택한다.

### 핵심 차별점

- 단순 정보 검색이 아니라 `다음 행동 추천`
- 추천에서 끝나지 않고 `예약 링크 연결`
- `보호자 공유 문구`와 `복지사 후속 안내문` 자동 생성
- 기관용 운영 화면에서 `추천-공유-참여-재추천` 루프 관리

### 기대효과

- 고립위험군의 첫 외출과 첫 참여 진입장벽 완화
- 보호자와 복지현장의 반복 안내 업무 절감
- 서울시 공공데이터의 정책 체감도 향상
- 자치구 및 기관 단위의 고립 대응 운영 효율화

## 3. 활용 데이터셋 표 초안

신청서와 상세기획서에는 아래처럼 `포털명 + 정확한 데이터셋 명칭` 형식으로 표기한다.

### 서울 열린데이터광장

- 서울시 노인여가복지시설 정보
- 서울시 문화행사 정보
- 서울시 공공서비스예약 정보
- 서울시 독거노인 현황

### 공공데이터포털

- 기상청 단기예보 조회서비스
- 에어코리아 대기오염정보 조회서비스

주의:

- 실제 제출 전에는 포털 화면상의 `정확한 데이터셋 명칭`으로 최종 통일
- 상세기획서와 신청서의 표기를 동일하게 맞출 것

## 4. AI 활용 설명 문구

잇다서울은 생성형 AI를 단순 문구 장식이 아니라 핵심 서비스 흐름에 사용한다.

- 사용자 상태 해석
  - 외출 가능성, 고립 위험, 보호자 개입 필요성 해석
- 추천 설명 생성
  - 왜 이 프로그램을 먼저 권하는지 자연어로 설명
- 보호자 공유 문구 생성
  - 문자/메신저 전달용 요약 메시지 생성
- 복지사 후속 안내문 생성
  - 체크인 문구와 재접촉 메시지 생성
