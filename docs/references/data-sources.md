# 잇다서울 데이터 소스 정리

## 1. 서울 열린데이터광장

역할:
고립위험 1인가구와 고령층에게 실제로 제안할 수 있는 복지·문화·예약 프로그램 후보를 구성한다.

현재 사용 대상 예시

- 독거노인 현황
- 노인여가복지시설
- 공공서비스예약 정보
- 문화행사 정보
- 문화행사 공공서비스예약 정보

활용 방식

- 자치구 기준 후보 탐색
- 복지형/문화형/예약형 프로그램 분류
- 장소, 일정, 요금, 접근성, 예약 가능 여부 반영
- 보호자 동행 가능성이나 고령친화성을 우선순위 계산에 반영

## 2. 공공데이터포털

역할:
오늘 당장 외출이 적절한지, 실내 중심이 좋은지, 짧은 이동이 나은지를 판단하는 도시 시그널을 제공한다.

현재 사용 대상 예시

- 기상청 단기예보 조회서비스
- 에어코리아 대기오염정보 조회서비스

활용 방식

- 실내/실외 우선순위 조정
- 이동 피로도 보정
- 보호자 동행이 필요한 날과 아닌 날의 판단 보조

현재 프로젝트에는 `data/scripts/fetch_city_signals.py`가 포함되어 있어 공공데이터포털 응답을 `city-signals.live.json`으로 변환할 수 있다.

## 왜 두 포털을 함께 써야 하는가

열린데이터광장만 쓰면 갈 수 있는 후보는 알 수 있지만 오늘 바로 움직이기 좋은지는 알기 어렵다.

공공데이터포털만 쓰면 오늘의 날씨와 대기질은 알 수 있지만 실제로 연결할 복지·문화·예약 후보군이 부족하다.

잇다서울은 두 포털을 결합해 아래 질문에 답한다.

> 지금 이 사람에게, 오늘 혹은 이번 주에, 가장 부담이 낮은 다음 연결 행동은 무엇인가?

## 현재 파이프라인 상태

- `fetch_seoul_open_data.py`
  - 서울 열린데이터광장 raw 수집
- `normalize_itda_programs.py`
  - raw 데이터를 `programs.normalized.json`으로 정리
  - 동시에 `programs.live.json`, `programs.live.meta.json` 생성
- `fetch_city_signals.py`
  - 공공데이터포털 도시 시그널을 `city-signals.live.json`으로 생성
